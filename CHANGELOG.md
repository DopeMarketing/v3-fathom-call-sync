All notable changes to V3 Fathom Call Sync will be documented in this file.

## [0.1.0] - 2026-03-27

### Added
- Initial project scaffold generated by Project Generator
- Complete Next.js 15 application structure with App Router
- Supabase integration with PostgreSQL database
- Database schema with 7 tables:
  - `users` - User account management
  - `oauth_connections` - OAuth token storage for external APIs
  - `sync_configurations` - User sync preferences and settings
  - `fathom_transcripts` - Cached transcript metadata from Fathom
  - `sync_jobs` - Background job tracking for sync operations
  - `sync_job_items` - Individual file transfer records within jobs
  - `uploaded_files` - Record of successfully uploaded files to Drive
- Row Level Security (RLS) policies for all database tables
- Site map implementation with 9 routes:
  - `/` - Public landing page
  - `/dashboard` - Main dashboard interface
  - `/connections` - OAuth connection management
  - `/history` - Sync job history and logs
  - `/settings` - User preferences and configuration
  - `/auth/fathom/callback` - Fathom OAuth callback handler
  - `/auth/google/callback` - Google OAuth callback handler
  - `/api/sync/trigger` - Manual sync trigger endpoint
  - `/api/sync/status` - Sync status checking endpoint
- Integration stubs for external services:
  - Google Drive API integration for file uploads
  - Fathom API integration for transcript fetching
  - n8n workflow automation support
  - Pipedream integration capabilities
  - Google Sheets integration for metadata tracking
- TypeScript configuration with strict mode enabled
- Tailwind CSS styling framework setup
- Development environment configuration
- Environment variables template (`.env.local.example`)
- Database migration files for all table schemas
- Supabase local development setup
- Complete documentation suite:
  - `README.md` - Project setup and usage instructions
  - `CLAUDE.md` - AI development briefing document
  - `TECHNICAL_DEBT.md` - Known shortcuts and improvement areas
  - `ROADMAP.md` - Feature development timeline
- Vercel deployment configuration
- Git repository initialization with proper `.gitignore`
- NPM package configuration with required dependencies
- Development scripts and build configuration
- Component directory structure for UI elements
- Business logic organization in `/lib` directory
- Server actions framework in `/actions` directory
- Database query layer in `/db` directory
- Type definitions for all data models
- Authentication system foundation with NextAuth.js

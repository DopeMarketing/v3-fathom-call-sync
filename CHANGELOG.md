# Changelog

All notable changes to the V3 Fathom Call Sync project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.1.0] - 2024-03-27

### Added

#### Project Foundation
- Initial project scaffold generated by Project Generator
- Next.js 15 application with App Router and TypeScript configuration
- Tailwind CSS setup with custom dashboard styling
- Supabase integration with PostgreSQL database
- Vercel deployment configuration

#### Database Schema
- Complete data model with 7 tables:
  - `users` table for user profiles and settings
  - `oauth_connections` table for storing API tokens securely
  - `sync_configurations` table for user sync preferences
  - `fathom_transcripts` table for cached transcript metadata
  - `sync_jobs` table for tracking sync operations
  - `sync_job_items` table for individual file sync status
  - `uploaded_files` table for Google Drive upload tracking
- Row Level Security (RLS) policies for all tables
- Database migrations and seed data structure

#### Application Routes
- Public landing page (`/`)
- Protected dashboard page (`/dashboard`)
- OAuth connections management (`/connections`)
- Sync history viewer (`/history`)
- User settings page (`/settings`)
- OAuth callback handlers:
  - Fathom authentication (`/auth/fathom/callback`)
  - Google Drive authentication (`/auth/google/callback`)
- API endpoints:
  - Manual sync trigger (`/api/sync/trigger`)
  - Sync status checker (`/api/sync/status`)

#### Integration Framework
- Fathom API client stub with TypeScript interfaces
- Google Drive API client stub with upload capabilities
- n8n workflow integration stub
- Pipedream automation integration stub
- Google Sheets API integration stub for metadata indexing

#### UI Components
- Basic dashboard layout with navigation
- Responsive design with Tailwind CSS
- Component structure for dashboard-heavy application
- Authentication wrapper components

#### Development Infrastructure
- TypeScript configuration with strict mode
- ESLint and Prettier setup
- Environment variable template (`.env.example`)
- Supabase CLI configuration
- Development server configuration

#### Documentation Suite
- Complete README.md with setup instructions
- CLAUDE.md AI development guide
- TECHNICAL_DEBT.md tracking known shortcuts
- ROADMAP.md with feature development plan
- This CHANGELOG.md for version tracking

#### Security & Authentication
- OAuth 2.0 flow stubs for Fathom and Google Drive
- Secure token storage in Supabase
- Row-level security policies for data protection
- Environment variable configuration for API keys
